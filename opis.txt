13:T1399,<h1>Opis Projektu</h1>
<h2>Wstęp</h2>
<p>Projekt <strong>GoBook</strong> ma na celu stworzenie aplikacji wieloplatformowej, która umożliwi użytkownikom tworzenie, zarządzanie i odczytywanie interaktywnych książek przygodowych (gamebook). Aplikacja będzie dostępna na urządzeniach mobilnych oraz w wersji webowej. Projekt opiera się na wykorzystaniu nowoczesnych technologii, takich jak <strong>React</strong> do tworzenia aplikacji webowej, <strong>Next.js</strong> do strony opisującej projekt, <strong>Spring Boot</strong> jako backend oraz <strong>Flutter</strong> do aplikacji mobilnej.</p>
<p>Projekt skupia się na bliskiej współpracy między członkami zespołu, którzy pracują nad różnymi komponentami systemu, zapewniając spójną integrację wszystkich technologii oraz ich funkcjonalności.</p>
<h2>Cel</h2>
<p>Głównym celem projektu <strong>GoBook</strong> jest stworzenie skalowalnej, responsywnej i intuicyjnej aplikacji, która umożliwi użytkownikom tworzenie interaktywnych książek przygodowych, które będą mogły być czytane zarówno w aplikacji mobilnej, jak i na stronie internetowej. Zespół dąży do zapewnienia doskonałej integracji między aplikacją mobilną, stroną internetową oraz backendem.</p>
<hr>
<h2>Architektura</h2>
<p>Projekt wykorzystuje podejście <strong>full-stack</strong> i jest podzielony na trzy główne komponenty:</p>
<h3>1. Aplikacja Mobilna</h3>
<ul>
<li><strong>Technologia</strong>: Flutter</li>
<li><strong>Główne funkcje</strong>:
<ul>
<li>Rejestracja i logowanie użytkowników</li>
<li>Tworzenie i edytowanie książek przygodowych</li>
<li>Odczytywanie i zarządzanie interaktywnymi książkami</li>
<li>Obsługa trybu offline i synchronizacja danych</li>
</ul>
</li>
</ul>
<h3>2. Backend</h3>
<ul>
<li><strong>Technologia</strong>: Java + Spring Boot</li>
<li><strong>Funkcje backendu</strong>:
<ul>
<li>Obsługa API REST do zarządzania danymi książek przygodowych</li>
<li>Bezpieczne uwierzytelnianie użytkowników (JWT)</li>
<li>Obsługa bazy danych relacyjnych (PostgreSQL)</li>
<li>Synchronizacja danych między aplikacjami mobilnymi i webowymi</li>
</ul>
</li>
</ul>
<h3>3. Strona Internetowa</h3>
<ul>
<li><strong>Technologia</strong>: React z Next.js</li>
<li><strong>Główne funkcje</strong>:
<ul>
<li>Responsywny interfejs użytkownika opisujący projekt i funkcjonalności</li>
<li>Interakcja z API backendu</li>
<li>Wsparcie dla lokalizacji i trybu ciemnego</li>
</ul>
</li>
</ul>
<hr>
<h2>Funkcjonalności</h2>
<p>Projekt zakłada wdrożenie poniższych funkcji:</p>
<ul>
<li><strong>Rejestracja i logowanie użytkowników</strong>:
<ul>
<li>Za pomocą e-maila lub mediów społecznościowych</li>
</ul>
</li>
<li><strong>Tworzenie i zarządzanie książkami przygodowymi</strong>:
<ul>
<li>Tworzenie, edytowanie, dodawanie treści i wybory w książkach przygodowych</li>
<li>Przeglądanie książek i historii użytkowników</li>
</ul>
</li>
<li><strong>Powiadomienia push</strong>:
<ul>
<li>W aplikacji mobilnej</li>
</ul>
</li>
<li><strong>Obsługa wielu języków</strong>:
<ul>
<li>Funkcjonalność lokalizacji (i18n) dla aplikacji webowej i mobilnej</li>
</ul>
</li>
</ul>
<hr>
<h2>Zespół</h2>
<p>Projekt realizuje zespół składający się z osób, które ściśle współpracują, aby zapewnić spójną integrację między komponentami aplikacji. Każdy członek zespołu odpowiada za różne obszary technologiczne i funkcjonalne projektu.</p>
